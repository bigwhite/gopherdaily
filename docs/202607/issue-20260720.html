Bun刚把Zig重写成Rust，这个团队却用487天反向重写
就在 Bun 团队宣布用 AI 花 11 天将 50 万行 Zig 代码重写为 Rust 之后，Roc 编译器团队发表了一篇方向截然相反的硬核复盘： 他们耗时 487 天，以纯人工方式将 30 万行的 Rust 编译器从零重写为了 Zig。重写的核心动机并非语言优劣，而是为了彻底解决一个因架构设计>导致的底层 Bug。在重新选择语言时，团队因构建速度、细粒度的内存控制、更贴合编译器生态的库，以及 Zig 对密集“不安全代码（Unsafe）”的>更好兜底，最终放弃了原有的 Rust。实测数据揭示了一个反直觉的结论：丢掉 Rust 借用检查器后，Zig 版编译器的内存损坏 Bug 数量不增反降， 且增量编译速度飙升至惊人的 35 毫秒。文章强调，在真实的软件工程中，没有绝对完美的语言，只有与特定架构约束、生态假设（如全局分配器 vs 显式 Arena）最契合的务实选择。

我开源了 cc-session-migrate ：让 Claude Code 会话在多台机器之间自由迁移
在使用大语言模型编程助手 Claude Code 时，开发者常面临会话数据受困于单一本地机器，无法在笔记本与远程服务器间自由切换>的痛点。为此，本文作者基于 Go 语言开源了一款命令行工具 csm (cc-session-migrate)。该工具采用 NAT 友好的 Hub-and-Spoke（中心辐射）WebSocket 架构，让用户无需配置繁琐的端口映射，即可实现跨机器的会话列表查看、无缝拉取与 claude --resume 续接。文章不仅分享了 5 分钟上 手指南和基于 S3 协议的自动备份功能，还复盘了开发过程中踩过的技术坑。csm 的发布，为重度多节点开发的高级极客补齐了 AI 编码时代最后一 块体验拼图。

从掌上设备的失败到AI时代的基石：Java官方纪录片，揭开一门语言30年的生死赌局
这篇基于 YouTube 官方最新长篇纪录片《The Java Story》的深度解读文章，首次全面揭秘了 Java 语言跌宕起伏的 30 年发展史>。文章讲述了 Java 最初作为一个濒临解散的掌上设备项目（Oak），如何意外搭上浏览器（Mosaic/Netscape）的快车一炮而红；揭示了其采用 C++ 语法的“伪装策略”以及无心插柳切入企业级服务端的传奇历程；披露了 J2EE 的灾难如何逼出了 Spring 与 Hibernate 这两把社区利剑，以及 Sun 与微软之间惊心动魄的世纪诉讼。文章还深度复盘了 Java 在失去创新的“停滞十年”后，如何通过 Oracle 的收购、Java 8 的豪赌（Lambda）以及 改为 6 个月一发布的敏捷节奏，最终完成惊天逆转。如今，面向高并发与 AI 时代，坐拥千万开发者的 Java 正凭借虚拟线程（Loom）、Valhalla 等底层革命再次焕发新生。这既是是一部技术演进史，也是一部充满妥协、抗争与重生的绝佳商业启示录。
