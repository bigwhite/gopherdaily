为了一个函数名，Go官方吵了两个月：maps.Same提案近日正式通过
Go 的 map 本质是指向哈希表的引用，但语言层面从未提供安全的方式判断“两个 map 变量是否指向同一份数据”——过去只能靠 reflect + unsafe 曲线救国。今天，Go 提案委员会正式采纳了 issue #78456，将在标准库 maps 包中新增 Same 函数。这个看似只有三行代码的提案，却在命名、nil 语义、NaN 陷阱、泛型签名四个维度上引发了长达数月的拉锯战，也让我们得以一窥 Go 语言演进背后严谨到近乎“较真”的设计哲学。
